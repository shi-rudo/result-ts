# Version 1 Migration

## Breaking Changes

- `matchErr()` handlers must now return a `Result` explicitly.
- `matchErr().when(...)`, `matchErr().whenGuard(...)`, and `matchErr().otherwise(...)` reject non-Result handler returns at runtime with `MatchErrHandlerNotResultError`.
- The package is promoted to the `1.0.0` major line.

## Migrating `matchErr()`

Before:

```ts docs-check:skip
result
    .matchErr()
    .when(ParseError, error => new ValidationError(error.message))
    .otherwise(error => new UnknownError(String(error)));
```

After:

```ts
import { err, ok } from '@shirudo/result';

class ParseError extends Error {}
class ValidationError extends Error {}
class UnknownError extends Error {}

const result = err<ParseError | UnknownError, string>(new ParseError('invalid'));

result
    .matchErr()
    .when(ParseError, error => err(new ValidationError(error.message)))
    .otherwise(error => err(new UnknownError(String(error))));
```

For recovery, wrap the recovered value in `ok(...)`:

```ts
import { Result, err, ok } from '@shirudo/result';

class NetworkError extends Error {}

const cachedValue = 'cached';
const result = Result.err<NetworkError | Error, string>(new NetworkError('timeout'));

result
    .matchErr()
    .when(NetworkError, () => ok(cachedValue))
    .otherwise(error => err(error));
```

## New APIs in v1

- `.matchError()` is the explicit name for Err-only fluent matching.
- `.match()` remains available as a compatibility alias.
- `.whenTag(key, value, handler)` supports discriminated-union error matching.
- `.matchErrorAsync()` and `.matchErrAsync()` provide async fluent matching.
